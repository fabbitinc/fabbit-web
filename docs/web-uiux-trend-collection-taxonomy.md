# 2026 웹 UI/UX 트렌드 수집 분류 체계

## 목적

2026년 웹 UI/UX 디자인 트렌드를 수집할 때, 항목이 지나치게 세분화되어 관리가 어려워지는 문제를 줄이기 위한 기준 문서다.

핵심 원칙은 다음과 같다.

- `대분류 + 상세분류 + 태그` 구조로 관리한다.
- 모든 항목을 1차 카테고리로 두지 않는다.
- 2단계 분류까지만 고정하고, 나머지는 태그로 교차 관리한다.

## 권장 구조

### 1차 대분류

1. Visual Language
2. Layout / Structure
3. Component Patterns
4. Interaction
5. Motion / Animation
6. Content / Brand Expression
7. UX / Usability
8. Technology / System

### 2차 상세분류

## 1. Visual Language

- 컬러 팔레트
- 그라디언트
- 타이포그래피
- 아이콘 / 일러스트
- 쉐도우 / 글로우 / 블러
- 텍스처 / 노이즈 / 유리감 / 메탈감

설명:
색, 질감, 표면감, 타이포 등 시각 언어 전반을 다루는 분류다. `그라디언트`, `쉐도우 스타일` 같은 항목은 이 영역에 포함한다.

## 2. Layout / Structure

- 그리드
- 섹션 구성
- 카드 배치
- 내비게이션 구조
- 밀도(여백 많음 / 촘촘함)
- 반응형 패턴

설명:
화면이 어떻게 조직되고 읽히는지에 대한 분류다. 페이지 구조, 콘텐츠 배치, 반응형 방식이 여기에 속한다.

## 3. Component Patterns

- 버튼
- 카드
- 입력폼
- 테이블 / 리스트
- 모달 / 드로어
- 탭 / 필터 / 칩
- 대시보드 위젯
- AI 인터페이스 컴포넌트

설명:
실제 UI를 구성하는 컴포넌트 패턴을 모아두는 분류다. `카드 스타일`, `컴포넌트 스타일`은 우선 이 영역에서 관리한다.

## 4. Interaction

- hover / focus / pressed
- drag & drop
- selection / multi-select
- command palette
- conversational UI
- 개인화 / adaptive UI

설명:
정적인 화면이 아니라 사용자의 입력에 반응하는 상호작용 방식을 분류한다.

## 5. Motion / Animation

- 페이지 전환
- 스크롤 모션
- 마이크로 인터랙션
- 로딩 / skeleton
- state transition
- 3D / parallax / physics

설명:
움직임 자체를 독립된 분류로 둔다. 인터랙션과 연결되지만, 수집 단계에서는 분리해야 검색성과 재사용성이 좋아진다.

## 6. Content / Brand Expression

- 전체 UI 스타일
- 톤앤매너
- 감성 키워드
- 비주얼 서사
- 브랜드 차별화 포인트

설명:
단순한 UI 패턴이 아니라 브랜드 성격과 메시지 표현 방식을 분류한다. 예: editorial, premium, playful, futuristic.

## 7. UX / Usability

- 온보딩
- empty / error / success 상태
- 접근성
- 가독성
- 신뢰감 / 설득 구조
- 검색 / 탐색 경험

설명:
사용성, 이해도, 신뢰감, 전환 경험처럼 실제 UX 품질에 직접 연결되는 항목을 관리한다.

## 8. Technology / System

- 디자인 토큰
- 다크모드
- 성능 친화 모션
- 생성형 AI 활용
- 멀티디바이스 적응
- 시스템화 가능성

설명:
트렌드가 단순 시각 요소가 아니라 기술적 구현 방식과 시스템 설계까지 확장될 때 사용하는 분류다.

## 카테고리와 태그를 분리하는 기준

카테고리는 "어디에 속하는가"를 정하는 용도다.
태그는 "어떤 성격인가"를 붙이는 용도다.

예를 들어 하나의 사례는 다음처럼 기록한다.

- 대분류: `Component Patterns`
- 상세분류: `Card`
- 태그: `editorial`, `high-contrast`, `soft-shadow`, `gradient-border`, `bento`, `dashboard`, `premium`

즉, `그라디언트`, `쉐도우`, `브랜드 무드`, `산업군`, `디바이스`, `구현 기술`까지 모두 카테고리로 승격하지 않고 태그로 교차 관리한다.

## 운영 원칙

### 1. 2단계까지만 고정한다

3단계 이상 강하게 고정하면 분류 충돌이 자주 생긴다.

예:

- 이 항목은 카드 스타일인가, 비주얼 스타일인가?
- 이 패턴은 인터랙션인가, 애니메이션인가?

따라서 `대분류 > 상세분류`까지만 고정하고, 나머지는 태그로 푼다.

### 2. 대분류는 6~8개 수준으로 유지한다

대분류가 많아질수록 탐색보다 분류 자체에 시간을 쓰게 된다.

### 3. 사례 저장 시 맥락 필드를 따로 둔다

각 사례에는 아래 정보를 함께 기록하는 것이 좋다.

- 왜 트렌드로 보이는지
- 어느 산업군에서 많이 보이는지
- 어떤 제품 유형에 적합한지
- 실제 구현 난이도는 어떤지
- 일시적 유행인지, 시스템화 가능한지

## 추천 저장 포맷

하나의 사례를 저장할 때는 아래 필드를 권장한다.

- 제목
- 대분류
- 상세분류
- 태그
- 한 줄 요약
- 왜 트렌드인가
- 대표 특징
- 적용하기 좋은 서비스 유형
- 주의할 점
- 참고 링크 / 레퍼런스

## 빠른 결론

- `크게 분류하고 상세분류할까?`에 대한 답은 `예`다.
- 다만 상세분류 아래를 더 잘게 쪼개기보다 `태그 체계`를 같이 설계하는 것이 중요하다.
- 추천 구조는 `대분류 + 상세분류 + 태그 + 맥락 필드`다.
