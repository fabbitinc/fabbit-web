### 도면 업로드

Part 상세에서 도면 올리기

이건 단순합니다. 해당 Part에 1:1 연결이니까 별도 매칭 로직 불필요.

Part 목록에서 여러 도면 올리기

이게 설계 포인트인데, 플로우를 정리하면:

도면 N개 업로드
↓
AI 분석 (도면별 품번 추출)
↓
┌─────────────────────────────────┐
│ 매칭 결과 리뷰 화면 │
│ │
│ ✅ 매칭됨: DWG-001 → BRK-001 │
│ ✅ 매칭됨: DWG-002 → BLT-001 │
│ ⚠️ 미매칭: DWG-003 → ??? │
│ ⚠️ 미매칭: DWG-004 → ??? │
└─────────────────────────────────┘
↓
미매칭 건에 대해: - 수동 매칭 (기존 Part 선택) - 신규 Part 생성 - 건너뛰기

확인할 점

1. 미매칭 시 "Part 생성해줄까?"는 좋은데, 자동 생성보다 사용자 확인 후 생성이 안전합니다. AI가 추출한 품번/품명을 보여주고 "이걸로 Part 만들까요?" 형태가 나을 듯. 의도치 않은 중복 Part 방지
   차원에서.
2. 1개 도면에 여러 Part가 포함된 경우 (조립도 등) — 이것도 대응해야 하는지? 아니면 1도면 = 1Part 전제로 갈 건지?
3. 기존 엑셀 업로드 플로우와의 관계 — 현재 Part 목록에서 "부품 업로드"(엑셀)가 이미 있는데, 도면 업로드도 같은 버튼 안에 탭으로 넣을 건지, 별도 버튼으로 뺄 건지?
