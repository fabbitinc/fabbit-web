## 초대 기반 회원가입 기능

### 개요
최고 관리자가 초대 링크를 생성하여 사용자를 초대하는 방식의 회원가입

### 기능 요구사항
- 관리자 페이지에서 초대 링크 생성
- 초대 링크에 만료 기간 설정 (예: 7일)
- 초대 링크로 접근 시 회원가입 폼 표시
- 회원가입 시 입력 항목:
  - 이름
  - 이메일 (초대 링크에 포함된 경우 자동 입력)
  - 비밀번호 / 비밀번호 확인
  - 부서 (선택)
- 소셜 계정 연동 옵션 (Google, Naver, Kakao)
- 초대 링크 사용 후 비활성화

### 구현 계획
1. 초대 토큰 검증 API 연동
2. `/invite/:token` 라우트 추가
3. 회원가입 폼 컴포넌트 (`SignupPage.tsx`)
4. 관리자 페이지에 초대 관리 기능

### API 엔드포인트 (예상)
- `POST /api/invites` - 초대 링크 생성
- `GET /api/invites/:token` - 초대 토큰 검증
- `POST /api/auth/signup` - 회원가입

## BOM Diff (버전 비교) 기능

### 개요
BOM 상세 화면에서 이전 버전과 현재 버전을 비교하여 변경사항을 시각화

### 기능 요구사항
- 버전 선택기 옆에 "비교 모드" 버튼 추가
- 두 버전(구버전/신버전) 선택 UI
- 변경사항 하이라이트:
  - 추가된 부품: 초록색 배경
  - 삭제된 부품: 빨간색 배경 (취소선)
  - 수정된 부품: 노란색 배경 (변경된 셀만)
- 변경 요약 표시 (추가 N개, 삭제 N개, 수정 N개)

### 구현 계획
1. BOM 버전 데이터 구조 설계 (mock data)
2. Diff 알고리즘 구현 (트리 비교)
3. 비교 모드 UI 컴포넌트
4. 하이라이트 스타일링

### 참고

## 트리 패널 + 버튼 UX 개선

### 문제
트리 패널 너비를 줄이면 + 버튼이 잘려서 보이지 않음

### 시도한 방법 (실패)
- `MIN_TREE_WIDTH`를 180px → 240px로 증가 (효과 없음)

### 근본 원인 (추정)
1. **Radix UI ScrollArea Viewport**: 가로 overflow가 hidden 처리됨
2. **노드의 paddingLeft**: 깊은 레벨(level 5 = 68px)일수록 콘텐츠가 오른쪽으로 밀림
3. **뷰포트 밖 요소**: shrink-0이어도 뷰포트를 벗어나면 잘림

### 추가 조사 필요
- ScrollArea의 horizontal scroll 활성화 검토
- `+` 버튼을 absolute position으로 오른쪽 고정하는 방안
- 노드 레이아웃 구조 자체 변경 (paddingLeft 대신 다른 방식)
- 호버 시 툴팁/팝오버로 버튼 표시하는 방안
