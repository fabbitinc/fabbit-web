# 05 전체 격차 전수 및 남은 이관 계획

## 작업 목표

- [x] `apps/web`와 `apps/web2`, `packages` 사이의 남은 화면/컴포넌트 격차를 전수 파악한다.
- [x] `web2` 컨벤션 위반 지점을 식별하고 정리 우선순위를 확정한다.
- [x] `web` 시각 기준으로 `packages`에 덮어써야 할 남은 UI/컴포넌트 대상을 확정한다.
- [x] 후속 단계의 병렬 작업 단위를 기능군별로 분해한다.

## 메모

- 이번 단계는 실제 이관 전에 남은 작업의 전체 범위를 닫는 전수 조사 단계다.
- `web`는 계속 읽기 전용으로 유지한다.
- 조사 결과는 이후 단계 문서의 작업 기준선으로 사용한다.
- 1차 전수 결과 기준 최우선 기능군은 `project-detail`, `parts`, `change-request`, `issue`, `change-management`, `dashboard`다.
- 현재 `web2`의 설정 화면 일부는 `packages/components`로 승격됐지만, 위 핵심 기능군의 screen 컴포넌트는 아직 hook/query/router와 화면 조합이 크게 섞여 있다.
- 후속 단계는 기능군별로 `packages/components` 화면 승격 -> `web2` 연결 래퍼 축소 -> storybook 검증 순으로 분해한다.
- 우선순위 1:
  - `apps/web2/src/features/project-detail/components/project-detail-screen.tsx`
  - `apps/web2/src/features/parts/components/part-detail-screen.tsx`
  - `apps/web2/src/features/parts/components/parts-list-screen.tsx`
  - `apps/web2/src/features/projects-list/components/projects-list-screen.tsx`
- 우선순위 2:
  - `apps/web2/src/features/change-management/components/change-management-screen.tsx`
  - `apps/web2/src/features/dashboard/components/dashboard-screen.tsx`
- 우선순위 3:
  - `apps/web2/src/features/change-request/components/change-request-detail-screen.tsx`
  - `apps/web2/src/features/issue/components/issue-detail-screen.tsx`
- 특히 우선순위 1, 2 기능군은 화면 조합 자체를 `packages/components`로 승격하기 쉬운 구조라 병렬 작업 대상으로 먼저 분리한다.
