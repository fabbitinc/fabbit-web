# 22. 패리티 자동검증 도입 및 잔차 구조 보정

## 목표

- `issue-detail` / `change-detail` 우측 패널 첨부파일 렌더링을 `apps/web` 원본 기준으로 다시 맞춘다.
- `parts-list` 상단 필터/액션 버튼 아이콘 폭을 `apps/web` 원본과 같은 마크업으로 복구한다.
- `web` / `web2` 동시 캡처와 diff threshold 검증을 한 번에 실행하는 자동검증 스크립트를 추가한다.

## 작업 메모

- 최신 동시 캡처 기준 잔여 diff는 `issue-detail 9987`, `parts-list 6020`, `change-detail 2935`다.
- `issue-sidebar`, `change-request-sidebar` 첨부파일 이름은 현재 `a[href]`로 렌더링되지만, `apps/web` 원본은 plain `p` 텍스트만 렌더링한다.
- `parts-list-screen`은 `size-3.5`, `size-4` 아이콘 클래스를 사용 중인데, `apps/web` 원본은 `h-3.5 w-3.5`, `h-4 w-4`를 사용한다.
- 육안 비교 대신 캡처 diff를 완료 기준으로 삼을 수 있도록 threshold 파일과 실행 스크립트를 함께 남긴다.

## 검증 항목

- `localhost:5173` / `localhost:5174` 동시 재캡처
- `issue-detail`, `change-detail`, `parts-list` diff 감소 확인
- `pnpm parity:web2`
- `pnpm --filter @fabbit/web2 exec tsc --noEmit`
- `pnpm lint:web2`
- `pnpm build:web2`
- `pnpm build-storybook`

## 작업 결과

- `packages/components/src/issue-sidebar.tsx`
  - 첨부파일 이름을 `a[href]`가 아니라 plain `p` 텍스트로 렌더링해 `apps/web` 우측 패널 마크업과 맞췄다.
- `packages/components/src/change-request-sidebar.tsx`
  - 첨부파일 이름 렌더링을 `apps/web` 원본과 같은 `p.truncate` 구조로 되돌렸다.
- `packages/components/src/parts-list-screen.tsx`
  - 검색/필터/내려받기 버튼 아이콘 클래스를 `size-*`에서 `h-* w-*`로 바꿔 `apps/web` 원본 폭 규칙과 맞췄다.
- `refactor/scripts/run-parity-threshold-check.mjs`
  - `web` / `web2` 동시 로그인 캡처, compare 실행, threshold 대조, `threshold-report.json` 생성까지 한 번에 처리하는 러너를 추가했다.
  - `pnpm parity:web2 -- ...` 형태도 동작하도록 `--` 인자를 무시하게 했다.
- `refactor/parity-thresholds.json`
  - 기본 캡처 호스트를 `test.lvh.me:5173` / `test.lvh.me:5174`로 두고, 화면별 diff threshold를 수치로 고정했다.
  - `localhost` 기본값에서는 로그인 후 리다이렉트가 안정적으로 완료되지 않아, 실제 통과 기준은 `test.lvh.me`를 사용한다.
- `package.json`
  - 루트에서 바로 `pnpm parity:web2`를 실행할 수 있도록 스크립트를 추가했다.

## 남은 격차

- 이번 단계로 육안 비교 없이도 `pnpm parity:web2` 하나로 회귀를 바로 걸 수 있게 됐다.
- 다만 threshold는 현재 실측값을 기준으로 잡은 회귀선이지, `web`와 픽셀 0 차이를 의미하지는 않는다.
- 현재 구조 기준 가장 큰 잔차는 `issue-create 4507`, `change-create 4115`, `parts-list 3731`, `issue-detail 3401` 순서다.
- 따라서 다음 단계가 필요하다면 기준은 “육안 확인”이 아니라, 이 네 화면의 threshold 자체를 더 낮추는 추가 구조 이관이어야 한다.
