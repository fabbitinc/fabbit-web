# 24. 매핑 분석 진입 및 자동 패리티 확장

## 목표

- `apps/web`의 `PartsTemplateAnalysisPage`, `PartsTemplateProcessingPage`, `PartsTemplateMappingPage`를 읽기 전용 기준으로 삼아 `packages/components`와 `apps/web2` 연결층의 남은 차이를 제거한다.
- `변경 요청 상세 > 변경 내용`과 매핑 플로우(분석 진입, 분석 진행, 매핑 확인)를 자동 패리티 캡처 대상에 포함한다.
- `apps/web2`는 hook/store/router 연결만 남기고, 화면 조합과 스타일은 `packages`가 소유하도록 정리한다.

## 메모

- `apps/web`는 수정하지 않는다.
- 기존 diff 스크립트는 `change-detail` 기본 탭과 일부 핵심 화면만 캡처하므로, 매핑 플로우 누락은 감지 실패가 아니라 캡처 범위 공백이다.
- 매핑 분석 진행 화면은 실제 업로드/상태 전이에 의존할 수 있으므로, 로컬 상태 기반 캡처나 전용 테스트 경로가 필요한지 함께 확인한다.

## 검증 항목

- `packages/components`의 매핑 분석/진행/확인 화면이 `apps/web` 구현 기준으로 갱신되었는가
- `apps/web2`의 매핑 관련 화면이 연결층만 남도록 축소되었는가
- 자동 패리티 캡처에 매핑 플로우와 `change-detail?tab=changes`가 포함되었는가
- `pnpm --filter @fabbit/web2 exec tsc --noEmit`
- `pnpm lint:web2`
- `pnpm build:web2`
- `pnpm build-storybook`
- `pnpm parity:web2`

## 작업 결과

- `packages/components/src/parts-template-analysis-screen.tsx`를 `apps/web/src/pages/parts/PartsTemplateAnalysisPage.tsx` 기준으로 다시 작성했다.
  - 업로드 카드, 형식 예시, 루트 지정 BOM 설명, 하단 안내 문구를 `web` 구조에 맞췄다.
  - 스타일 책임은 기존 `parts-template-mapping-screen.css`를 재사용해 `packages/components`가 소유한다.
- `apps/web2/src/features/part-template-mapping/components/parts-template-analysis-screen.tsx`는 업로드/라우팅 연결만 남기고, 파일 개수 제한과 지원 포맷 경고를 `web`과 같은 동작으로 복구했다.
- `packages/components/src/parts-template-processing-screen.tsx`를 `apps/web/src/pages/parts/PartsTemplateProcessingPage.tsx` 기준으로 다시 작성했다.
  - 헤더, 진행률, 처리 단계, 로그 박스, 업로드 경고 UI를 `web` 구조에 맞췄다.
- `apps/web2/src/features/part-template-mapping/components/parts-template-processing-screen.tsx`는 처리 단계 시퀀스를 `web` 흐름으로 다시 맞췄다.
  - parsing 선행 단계
  - 재시도 시 기존 업로드 재사용 로그
  - AI 분석 로그 애니메이션
  - 최소 애니메이션 시간 확보로 진행률이 실제로 보이도록 조정
- `packages/components/src/parts-template-mapping-board.tsx`는 `apps/web/src/features/mapping/components/kanban/*` 구조 기준으로 다시 정렬했다.
  - 카드 레이아웃을 1줄 구조로 되돌리고 `source -> target` 인라인 표시 복구
  - 선택기 폭/라벨/툴팁/매칭키 뱃지/관계속성 뱃지 복구
  - 샘플 미리보기/컬럼 헤더/빈 상태/드래그 오버레이를 `web` 기준으로 보정
- `apps/web2/src/features/part-template-mapping/lib/template-mapping-property-labels.ts`를 추가하고, `apps/web`의 `locales/ko/mapping.ts` 기준 속성 표시명을 `web2` 연결층에서 주입하도록 정리했다.
- `refactor/scripts/capture-authenticated-routes.mjs`를 확장했다.
  - `change-detail?tab=changes`
  - `/parts/templates`
  - `/parts/templates/processing`
  - `/parts/templates/mapping`
  - 위 네 구간을 실제 로그인/업로드/처리 흐름으로 캡처한다.
- `refactor/fixtures/template-analysis-sample.csv`를 추가해 매핑 플로우 자동 캡처에 쓰는 고정 fixture를 만들었다.
- `refactor/parity-thresholds.json`에 새 라우트 threshold를 추가했다.
  - `parts-template-analysis.png`
  - `parts-template-processing.png`
  - `parts-template-mapping.png`
  - `change-detail-changes.png`

## 메모 업데이트

- 이번 단계로 사용자 지적이던 “매핑 확인 화면 다름 / 매핑 분석 진행 프로그레스 동작 안함 / 매핑 분석 진입화면 다름 / diff 스크립트가 감지 못함”은 구조와 자동 검증 양쪽에서 처리했다.
- 남은 패리티 차이는 여전히 다른 화면들의 기존 threshold 안 잔차들이다. 다만 매핑 플로우와 변경 내용 탭은 이제 자동 캡처 범위 밖이 아니다.
