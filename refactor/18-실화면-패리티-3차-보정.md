# 18. 실화면 패리티 3차 보정

## 목표

- `parts-list`의 기본 화면 지오메트리와 테이블 표현을 `apps/web`의 `PartsTableContent` 기준으로 다시 맞춘다.
- `issue detail` / `change detail`의 루트 간격과 탭 underline 마크업을 `apps/web` 기준으로 정리한다.
- `dashboard`의 usage 섹션 노출 조건을 `apps/web`와 동일하게 맞춘다.
- 리치 텍스트 본문/댓글이 문자열로 내려오는 경우도 `web`와 동일하게 표시되도록 경계를 보강한다.

## 작업 메모

- 최신 캡처 diff 기준 큰 화면은 `parts-list`, `change-detail`, `app-shell(=dashboard 포함)`, `issue-detail` 순이다.
- `apps/web/src/pages/projects/ChangeRequestDetailPage.tsx`의 루트는 `mx-auto max-w-[1160px]`이며, 현재 `packages/components` 상세 화면은 추가 `px-6 py-6`가 있어 좌우/상하 여백이 더 크다.
- `apps/web/src/components/parts/PartsTableContent.tsx`는 현재 `packages/components` parts list보다 컬럼 폭, revision badge, lifecycle badge, empty/loading 구조가 더 촘촘하다.
- `apps/web/src/pages/dashboard/DashboardPage.tsx`는 `currentMembership.role`이 `ADMIN` 또는 `OWNER`일 때만 usage 섹션을 노출한다.

## 검증 항목

- `test.lvh.me:5173` / `test.lvh.me:5174` 재캡처 및 diff 재확인
- `pnpm lint:web2`
- `pnpm build:web2`
- `pnpm build-storybook`

## 작업 결과

- `packages/components/src/parts-list-table.tsx`를 `apps/web/src/components/parts/PartsTableContent.tsx` 기준으로 다시 정리해 컬럼 폭, badge, empty/loading, footer 문구를 맞췄다.
- `packages/components/src/issue-detail-screen.tsx`, `packages/components/src/change-request-detail-screen.tsx`의 루트 여백과 탭 underline 마크업을 `apps/web` 기준으로 보정했다.
- `packages/components/src/dashboard-screen.tsx`에서 usage 섹션 노출 조건과 stats 표시를 `apps/web` 원본 조건으로 되돌렸다.
- `apps/web2/src/lib/rich-text.ts`와 `apps/web2/src/features/issue/components/issue-detail-screen.tsx`를 보강해 문자열 본문/댓글도 `web`처럼 rich text 문서로 정규화되도록 맞췄다.
- `packages/ui/src/tiptap-editor.tsx`에 read-only 본문/댓글 padding을 보강해 상세 화면 body/comment 높이 차이를 줄였다.

## 검증 결과

- 이 단계 이후 `app-shell`, `change-detail`, `issue-detail`, `parts-list` diff가 모두 감소했다.
- 후속 19단계 직전 기준으로 `parts-list`와 `project-list`가 남은 핵심 격차로 정리됐고, `issue-detail`은 body/timeline 높이 차이가 크게 줄어든 상태가 됐다.

## 남은 격차

- `project-list` footer pagination이 한때 `safePage / totalPages`로 단순화돼 `apps/web` 원본과 다시 어긋났고, 이 항목은 19단계에서 복구했다.
- `issue-detail`은 read-only Tiptap 외에도 activity detail 포맷 매핑 차이가 남아 있었고, 이 항목도 19단계에서 추가 정리했다.
