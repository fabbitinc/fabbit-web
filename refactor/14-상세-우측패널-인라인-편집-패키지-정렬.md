# 14 상세 우측패널 인라인 편집 패키지 정렬

## 작업 목표

- [x] `packages/components`의 변경 요청 상세 화면 우측 패널을 `apps/web` 기준 인라인 picker/linked issue/attachment 구조로 재구성한다.
- [x] `packages/components`의 이슈 상세 화면 우측 패널을 `apps/web` 기준 인라인 picker/linked change/attachment 구조로 재구성한다.
- [x] `apps/web2` 상세 wrapper에서 `DetailSelectionDialog` 중심 상태를 제거하고 lookup/search/sync props만 전달하도록 축소한다.
- [x] 댓글 액션 영역과 not-found/error 분기를 `web` 동작 기준에 맞춘다.

## 작업 메모

- `apps/web`는 읽기 전용이다.
- `packages/components`는 props-only 조합물이어야 하므로 inline 편집 상태는 화면 내부 UI 상태까지만 가지고, 실제 저장은 callback props로 위임한다.
- `apps/web2`는 lookup query, sync mutation, router 연결만 남기고 상세 UI 상태를 직접 소유하지 않는다.
- 첨부파일 섹션은 업로드 스피너, hover 삭제 affordance, 파일 수 표시를 `web` 기준으로 맞춘다.
- `packages/components/src/change-request-sidebar.tsx`, `packages/components/src/issue-sidebar.tsx`는 기존 정적 카드/외부 dialog 의존을 걷어내고 picker section + linked entity popover + attachment hover 삭제 구조로 재구성했다.
- `packages/components/src/change-request-detail-screen.tsx`, `packages/components/src/issue-detail-screen.tsx`는 헤더 제목 편집 위치, 댓글 하단 액션, `createdAt` 기준 타임라인 라벨, not-found/error 분리를 `web` 기준으로 맞췄다.
- `apps/web2/src/features/change-request/components/change-request-detail-screen.tsx`, `apps/web2/src/features/issue/components/issue-detail-screen.tsx`는 enable/search state와 mutation 연결만 남기고 `packages/components` public props를 소비하도록 축소했다.
- 공용 타임라인의 `외 N개` 표시는 `packages/components/src/timeline-event.tsx`에서 Tooltip으로 보강했다.
- 잔여 위험은 실제 브라우저에서 `web` 대비 픽셀/spacing 차이를 눈으로 다시 보는 단계다. 구조적 이관과 빌드 검증은 끝냈지만, 최종 시각 패리티는 수동 화면 비교가 한 번 더 필요하다.
