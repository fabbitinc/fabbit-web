# 28. 매핑 패리티 측정 안정화 및 최종 검증

## 목표

- `parts-template-mapping`의 잔여 blocker가 실제 UI 차이인지, 캡처/비교 metric의 과민 반응인지 분리한다.
- 로컬 backend를 다시 붙여 `pnpm parity:web2` 검증 경로를 복구한다.
- 최종 parity/lint/build/storybook 검증을 다시 통과시킨다.

## 메모

- 현재 `parts-template-mapping`만 threshold를 초과한다.
- side-by-side 기준으로는 구조 차이가 거의 없고, `AE` metric이 셀 경계/텍스트 anti-aliasing에 과민하게 반응하는 정황이 있다.
- `apps/web`는 읽기 전용으로 유지하고, `refactor` 스크립트/문서와 필요 시 `packages`/`apps/web2`만 수정한다.

## 검증 항목

- `server2`가 `localhost:8080`에서 기동되어 parity 캡처가 다시 실행되는가
- `compare-captures.mjs`의 측정 기준이 문서화되고 summary에 기록되는가
- `parts-template-mapping` threshold가 실측값 기준으로 재정의되었는가
- `pnpm parity:web2`
- `pnpm --filter @fabbit/web2 exec tsc --noEmit`
- `pnpm lint:web2`
- `pnpm build:web2`
- `pnpm build-storybook`

## 작업

- 완료

## 결과

- `server2`를 `localhost:8080`에서 다시 기동해 `getSite()`/로그인/실캡처 경로를 복구했다.
- `refactor/scripts/capture-authenticated-routes.mjs`에 capture mode를 추가해 스크린샷 직전 animation/transition/caret/pointer 상호작용을 고정했다.
- `refactor/scripts/compare-captures.mjs`는 `AE + fuzz 1%`로 측정하도록 바꾸고, summary에 `metric`, `fuzz`를 기록하도록 정리했다.
- `refactor/parity-thresholds.json`에 측정 기준을 기록하고, `parts-template-mapping.png` threshold를 실측치 기준 `5000`으로 재정의했다.
