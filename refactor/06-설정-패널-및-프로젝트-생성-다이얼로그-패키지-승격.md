# 06 설정 패널 및 프로젝트 생성 다이얼로그 패키지 승격

## 작업 목표

- [x] `web2` 로컬 설정 패널 중 store 연결만 분리하면 되는 화면을 `packages/components`로 승격한다.
- [x] 프로젝트 생성 다이얼로그를 `packages/components`로 승격하고 `web2`는 store/action 연결만 남긴다.
- [x] 신규 패키지 컴포넌트 export와 storybook 스토리를 추가한다.
- [x] `web2` 연결 래퍼를 정리한 뒤 빌드/린트/스토리북으로 검증한다.

## 메모

- 이번 단계 대상은 `organization security`, `user notifications`, `user preferences`, `create-project-dialog`이다.
- `packages/components`는 props-only를 유지하고 store/action/router 의존을 가져오지 않는다.
- `web2`는 Zustand/action 연결과 토스트 처리만 담당한다.
- 추가된 패키지 컴포넌트:
  - `packages/components/src/project-create-dialog.tsx`
  - `packages/components/src/organization-security-settings-panel.tsx`
  - `packages/components/src/user-notifications-settings-panel.tsx`
  - `packages/components/src/user-preferences-settings-panel.tsx`
- `web2` 래퍼 정리:
  - `apps/web2/src/features/projects-list/components/create-project-dialog.tsx`
  - `apps/web2/src/features/organization-settings/components/organization-security-settings-tab.tsx`
  - `apps/web2/src/features/user-settings/components/user-notifications-settings-tab.tsx`
  - `apps/web2/src/features/user-settings/components/user-preferences-settings-tab.tsx`
- storybook 스토리를 추가해 신규 패키지 컴포넌트의 독립 렌더링을 확인할 수 있게 했다.
- 검증 결과:
  - `pnpm build:web2` 성공
  - `pnpm lint:web2` 성공
  - `pnpm build-storybook` 성공
- storybook과 `web2` 빌드의 번들 크기 경고는 계속 남아 있지만 실패 원인은 아니다.
